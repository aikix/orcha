The Problem Orcha Solves

Without Orcha

  • Agent re-discovers workspace topology every session
  • New developer takes 2 days to set up locally
  • "Which services depend on this?" — ask a human
  • PR review misses cross-service breakage
  • Service goes down, nobody notices
  • Tribal knowledge in Slack threads and people's heads

With Orcha

  • Agent knows services, deps, and topology instantly
  • /orcha-onboard — productive in 15 minutes
  • /orcha-impact — concrete blast radius
  • /orcha-pr-review — traces cross-repo impact
  • orcha watch --restart — auto-recovers
  • orcha.config.yaml — versioned, shared, declarative

Built for Agents, Used by Developers

🔍

Auto-Discovery

Scan a GitHub org or local workspace. Reads actual source code to detect ports, health endpoints, dependencies. Supports Node.js, Python, Go.

🎯

Profile-Aware

--profile staging swaps the entire dependency graph. UI devs skip Docker entirely. Backend devs get full local infra.

📊

Blast Radius

orcha impact shows direct dependents, transitive dependents, affected probes, and affected flows. Know before you break.

🤖

MCP Server

Any MCP-compatible agent gets workspace context automatically. Resources for topology, health, KB. Tools for config, impact, search.

📚

Living Knowledge Base

KB grows naturally from PR reviews. /orcha-kb-baseline bootstraps, /orcha-kb-update keeps it fresh.

🛡

Health Monitoring

orcha watch --restart continuously monitors services and auto-restarts on failure. JSON event stream for agent consumption.

Get Started

# Install
curl -fsSL https://raw.githubusercontent.com/aikix/orcha/main/install.sh | bash

# Or from source
git clone https://github.com/aikix/orcha.git && cd orcha
bun install && bun link

# Initialize your workspace
/orcha-onboard https://github.com/your-org
